# 架构说明

## 项目边界

Retail Scrapers 是提取层，不是价格监控平台。核心只处理四件事：

1. 接收渠道和筛选参数。
2. 调用渠道 adapter 获取公开商品数据。
3. 校验结果是否完整、合理和可追踪。
4. 输出统一的目录或价格记录。

项目不提供数据库、商品主数据、汇率、告警、前端或私有数据同步。

## 运行流程

```text
CLI / Python API
      ↓
registry 选择渠道 adapter
      ↓
adapter 选择 API、HTTP 或 Playwright
      ↓
统一 CatalogRecord / PriceRecord
      ↓
validation 严格校验
      ↓
CSV / JSONL / Python 对象
```

## 目录职责

- `core/`：浏览器、价格解析等跨渠道基础能力。
- `adapters/`：网站特定的请求、分页和字段映射。
- `models.py`：稳定的公共输出契约。
- `validation.py`：与渠道无关的质量底线。
- `runner.py`：Python API 和运行时生命周期。
- `cli.py`：命令行参数，不包含抓取逻辑。

## 设计原则

- API 优先，但不假设接口永远稳定。
- 页面适配器必须与其它渠道隔离。
- 渠道失败返回非零状态，不把半量伪装成成功。
- 用户数据只从输入流向输出，不写入仓库。
- 渠道常量只能描述公开站点，不包含使用者的业务规则。
